Day 6: API의 이해와 requests 라이브러리
API(Application Programming Interface)는 에이전트가 외부 서비스(날씨 정보, 구글 검색 등)와 대화하기 위한 '메뉴판' 또는 '전화기'입니다. requests는 Python에서 이 '전화'를 걸게 해주는 가장 표준적인 라이브러리입니다.

1) 왜 필요한가?
에이전트가 "서울 날씨 어때?"라는 질문을 받으면, 스스로 날씨를 알 수 없습니다.

반드시 외부 '날씨 API'에 전화를 걸어(Request) "서울 날씨 데이터 줘"라고 요청해야 합니다.

API 서버는 날씨 데이터를 JSON 형태로 응답(Response)해 줍니다.

requests 라이브러리가 이 '전화 거는' 행위를 수행합니다.
